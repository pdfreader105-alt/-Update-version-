# المعجم الثلاثي — GitHub Pages এ ডিপ্লয় করার নির্দেশিকা

এই ফোল্ডারে একটা সম্পূর্ণ, ইনস্টলযোগ্য PWA (Progressive Web App) আছে — কোনো বিল্ড স্টেপ লাগবে না, সরাসরি GitHub Pages-এ আপলোড করলেই কাজ করবে।

## ফাইল গঠন

```
index.html         → মূল পেজ
style.css           → সব স্টাইল
app.js               → অ্যাপের লজিক
data.js              → ১৭৯৩টি শব্দের ডেটা
manifest.json        → PWA কনফিগারেশন
service-worker.js    → অফলাইন সাপোর্ট + ইনস্টলযোগ্যতা
icons/                → সব আইকন (৪৮px থেকে ৫১২px + maskable + favicon)
```

**সবগুলো ফাইল ও ফোল্ডার একই লেভেলে, রিপোর রুটে (বা যেই সাবফোল্ডারে সাইট রাখবেন) থাকতে হবে — গঠন এলোমেলো করবেন না।**

## ধাপে ধাপে ডিপ্লয়

### ১. রিপো তৈরি করুন (বা আগেরটা ব্যবহার করুন)
GitHub-এ নতুন রিপো বানান, অথবা আগের রিপোর কন্টেন্ট মুছে এই ফাইলগুলো দিয়ে replace করুন।

### ২. ফাইল আপলোড করুন — **git ব্যবহার করাই সবচেয়ে নিরাপদ**
GitHub-এর ওয়েব ইন্টারফেস দিয়ে ফোল্ডার আপলোড করলে প্রায়ই সমস্যা হয় (যেমন আপনার আগের `icons` ফোল্ডারের সমস্যা) — কারণ ব্রাউজার থেকে ড্র্যাগ-ড্রপ করলে মাঝেমধ্যে নেস্টেড ফোল্ডার ঠিকমতো commit হয় না বা ফাইলের কেস/এক্সটেনশন এলোমেলো হয়ে যায়।

**সবচেয়ে নির্ভরযোগ্য উপায় (Terminal থেকে):**
```bash
git clone https://github.com/আপনার-ইউজারনেম/আপনার-রিপো.git
cd আপনার-রিপো
# এই ফোল্ডারের সব ফাইল এখানে কপি করুন (icons ফোল্ডারসহ)
git add .
git commit -m "PWA: manifest, service worker ও আইকন যোগ করা হলো"
git push
```

**যদি ওয়েব ইন্টারফেস দিয়েই করতে চান:**
- "Add file" → "Upload files" এ যান
- পুরো `icons` ফোল্ডারটা একসাথে ড্র্যাগ করে ছাড়ুন (আলাদা আলাদা ফাইল না দিয়ে পুরো ফোল্ডার ড্র্যাগ করুন — GitHub আধুনিক ব্রাউজারে ফোল্ডার গঠন ঠিক রাখে)
- আপলোডের পর অবশ্যই রিপোতে গিয়ে চেক করুন যে `icons/icon-192.png` ও `icons/icon-512.png` সত্যিই আছে এবং ক্লিক করলে ছবি দেখা যাচ্ছে (ফাইল সাইজ 0 না)

### ৩. GitHub Pages চালু করুন
রিপোর **Settings → Pages** এ গিয়ে:
- Source: "Deploy from a branch"
- Branch: `main` (বা যেটা ব্যবহার করছেন), ফোল্ডার: `/ (root)`
- Save করুন

কয়েক মিনিট পর আপনার সাইট চালু হবে: `https://আপনার-ইউজারনেম.github.io/আপনার-রিপো/`

### ৪. যাচাই করুন
মোবাইল Chrome দিয়ে সাইটে যান, তারপর ডেস্কটপ Chrome দিয়েও দেখুন:
- ডেস্কটপ Chrome-এ **DevTools (F12) → Application → Manifest** ট্যাবে গিয়ে দেখুন কোনো error/warning আছে কিনা
- একই ট্যাবে **Service Workers** সেকশনে দেখুন worker "activated and running" দেখাচ্ছে কিনা
- সব ঠিক থাকলে অ্যাড্রেস বারের পাশে একটা ইনস্টল আইকন (⊕) দেখা যাবে, অথবা মোবাইলে "হোম স্ক্রিনে যোগ করুন" ব্যানার আসবে

## আগের ইনস্টলযোগ্যতা সমস্যার কারণ কী ছিল হতে পারে
Android Chrome-এ PWA ইনস্টল না হওয়ার সবচেয়ে সাধারণ কারণগুলো এই ভার্সনে ঠিক করা হয়েছে:
- **আপেক্ষিক পাথ (relative path)** — `manifest.json`-এ `start_url` ও আইকন পাথ `./` দিয়ে শুরু, তাই GitHub Pages-এর সাবফোল্ডারে (`/repo-name/`) হোস্ট হলেও ভাঙবে না
- **সঠিক সাইজের আইকন** — 192×192 ও 512×512 উভয়ই আছে, প্লাস `maskable` পার্পাসের আইকনও আলাদাভাবে দেওয়া আছে (Android-এর জন্য জরুরি)
- **সক্রিয় Service Worker** — Chrome ইনস্টলযোগ্যতার জন্য একটা রেজিস্টার্ড ও কার্যকর service worker আবশ্যক করে, এটা `app.js`-এ স্বয়ংক্রিয়ভাবে রেজিস্টার হয়
- **HTTPS** — GitHub Pages এমনিতেই HTTPS দেয়, তাই এটা নিয়ে চিন্তা নেই

যদি তারপরও ইনস্টল বাটন না আসে, DevTools-এর Manifest ট্যাবে ঠিক কোন এরর দেখাচ্ছে সেটা জানালে আমি ঠিক করে দিতে পারব।

## নতুন শব্দ যোগ করতে চাইলে
`data.js` ফাইলে একটা JavaScript অ্যারে আছে, প্রতিটা এন্ট্রি এভাবে:
```js
{"i":1,"a":"اُنْظُرْ","b":"তাকাও","e":"Look","u":"ভূমিকা ও নির্দেশাবলি"}
```
`i` = ইউনিক আইডি, `a` = আরবি, `b` = বাংলা, `e` = ইংরেজি, `u` = ইউনিটের নাম। নতুন এন্ট্রি যোগ করে ফাইলটা সেভ করে আবার push করলেই যথেষ্ট, কোনো বিল্ড লাগবে না।
